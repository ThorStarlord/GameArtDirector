# Readability Rules

All prompts should optimize for gameplay readability, not just aesthetic richness.

## General rules

- Keep the primary silhouette easy to parse.
- Preserve one dominant read before secondary details.
- Use contrast to separate key forms.
- Avoid detail density that breaks at gameplay scale.
- Make the asset readable in the likely in-game camera distance.

## Technical framing

- Mention transparent background when the asset should be extracted.
- Mention clean isolation when the subject must be cut out easily.
- Mention padding for icons and UI-facing assets.
- Mention modularity and scalability for UI pieces.
- Mention tiling only when environment pieces benefit from it.

## By asset category

### Icons

Include: transparent background, single centered subject, high readability at 64x64 pixels, strong silhouette.

### Sprites

Include: clean outline, consistent perspective, game-ready proportions, transparent background.

### UI Elements

Include: modular design, scalable layout, readable at gameplay resolution.

### Environment Pieces

Include: tileable if appropriate, consistent lighting, separate foreground and background elements.
