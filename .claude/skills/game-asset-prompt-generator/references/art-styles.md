# Art Styles

Infer the minimum style direction needed for a strong result.

## Style handling rules

- Keep style assumptions neutral unless the user implies a genre, setting, or project-specific look.
- Favor concise art-direction language over genre-stacking.
- If the user names a project style, continue that visual language instead of inventing a new one.
- Describe what the asset should feel like in-game, not just what it should look like in isolation.

## Useful style dimensions

- Stylized or semi-stylized
- Painterly or graphic
- Clean fantasy, grounded fantasy, sci-fi, horror, cozy, arcade, or tactical
- Soft, rounded, friendly shapes versus angular, dangerous, severe shapes
- Flat-color readability versus material-rich rendering

## Modern model guidance

Favor:
- clear descriptive language
- visual intent
- art direction
- composition
- gameplay readability
- asset production requirements

Avoid:
- masterpiece
- best quality
- 8k
- award winning
- trending on artstation
